# Threat Model Assessment

## System Boundaries
- Entry points: CLI tools, prompts, agent workflow scripts.
- Data storage: Local configuration files, `.agents` directory, Git history.

## STRIDE Evaluation
### Spoofing
- Caller identity is validated via Pydantic schemas for tool inputs.
- No unauthenticated external calls.

### Tampering
- All input parameters are validated; no direct string parsing.
- No mutable global state without explicit checks.

### Repudiation
- Git commit hooks provide audit trail; pre‑commit logs all changes.
- No silent file modifications.

### Information Disclosure
- No hard‑coded secrets; API keys are scanned by Semgrep.
- No stack traces are printed in production hooks.

### Denial of Service
- Rate limits are enforced by pre‑commit (no heavy operations during commit).
- No long‑running shell commands allowed (blocked by hooks).

### Elevation of Privilege
- Tool actions require explicit permission checks via custom hooks.
- No privileged commands are exposed to untrusted agents.
